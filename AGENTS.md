# 网站改造总手册

## 文件定位

这份 `AGENTS.md` 是当前根目录的总工作手册，用于指导 AI 在这个目录下处理各类客户网站改造项目时的默认工作方式。

它不是某一个客户项目的专属说明，而是所有客户文件夹共用的顶层规则。后续每个客户都会有自己的独立文件夹；如果某个客户有特殊要求，可以在该客户文件夹内补充单独说明，且客户级说明优先于本手册。

此外，所有客户项目除了在本地根目录下拥有各自独立文件夹外，也应同步整理并上传到 GitHub 仓库 `stoney6379/pages` 中，并在该仓库内保持同样的按客户文件夹划分方式。

## 适用范围

本手册默认适用于以下类型的网站项目：

- 单页面展示站
- 品牌展示站
- 本地门店官网
- 服务介绍型官网
- 小型企业官网
- 个人品牌或工作室展示页

常见行业包括但不限于：

- 美容院
- 医美
- SPA
- 美发
- 健身
- 餐饮
- 本地服务业
- 工作室
- 小型品牌展示站

这类项目的核心通常是展示、品牌升级、信息整理和转化提升，而不是复杂业务系统开发。

## 默认项目目标

AI 在处理这类项目时，默认目标不是简单“把页面做出来”，而是完成以下升级：

- 提升原网站的视觉质感和品牌感
- 提升页面的专业感、信任感和真实感
- 优化信息结构，让访客更容易理解服务与卖点
- 提升咨询、预约、到店、留资等轻转化能力
- 在桌面端和移动端都达到可直接使用的展示效果

## 默认不做的功能范围

除非客户明确提出，否则这类项目默认不包含以下内容：

- 支付系统
- 购物车
- 电商结账流程
- 会员系统
- 登录注册
- 复杂后台
- 多角色权限系统
- 复杂业务数据流

如果客户确实提出上述需求，应当先单独确认是否超出当前项目边界，而不是默认自行加入。

## 开始每个客户项目时的默认流程

AI 进入任意客户文件夹后，应按以下顺序工作：

1. 先主动向用户提问关键背景信息
2. 为该客户创建独立项目文件夹，并保持本地与 GitHub 仓库中的文件夹命名一致
3. 确认客户行业、原网站、目标风格、目标受众和特殊要求
4. 调研原网站当前的信息结构、视觉问题和已有素材
5. 调研该行业的高质量头部网站，并拆解其设计语言
6. 调研 Google Maps、客户公开社媒或其他公开渠道中的真实门店照片
7. 在真实业务背景和真实素材基础上完成新版展示站方案与实现
8. 及时将该客户文件夹内的代码同步上传到 GitHub 仓库 `stoney6379/pages` 对应文件夹中

默认不要在信息未明确时直接开始写页面。

## 每次必须先问的关键问题

在开始做某个客户项目时，AI 默认应主动向用户确认以下问题：

- 这个客户是什么行业
- 原网站是什么，或者原网站链接是什么
- 这次改造想升级成什么样的气质或风格
- 目标用户是谁
- 有没有明确参考的网站、品牌或视觉方向
- 有没有必须保留的文案、结构、图片或品牌元素
- 客户是否有 Google Maps 页面、社媒主页或真实门店照片
- 有没有特殊要求会覆盖默认流程

如果用户已经在项目说明中提供了其中一部分信息，AI 应先消化现有内容，再补问真正缺失的关键项，避免重复提问。

## 原网站调研要求

对于已有旧站的客户，AI 默认应先研究原网站，而不是跳过现状直接重做。

调研重点包括：

- 现有页面的信息结构
- 当前服务项目和卖点表达
- 品牌气质是否过时、廉价、混乱或缺乏信任感
- 图片是否失真、老旧、虚假感强或与品牌不匹配
- CTA 是否弱、模糊或缺失
- 移动端是否落后或体验不完整

AI 应输出对原网站问题的判断，并据此决定新版页面该强化哪些部分。

需要特别注意的是：原网站的主要参考价值，在于提取真实业务信息与真实渠道信息，例如服务内容、品牌名称、联系方式、Google Maps、Instagram、Facebook 等原始链接，而不是延续其旧的视觉风格。

除非用户明确要求保留某些品牌元素，否则 AI 不应把原网站的配色、版式、审美、氛围、组件风格或整体美术方向作为新版设计依据。旧站如果本来就不够高级、过时或缺乏设计感，那么新版的意义正是完成审美升级，而不是换一种方式重复旧站风格。

## 行业参考要求

AI 默认必须参考目标行业中高质量、审美成熟、品牌完成度高的网站，不应只依赖模型自己的默认审美。

参考时应遵循以下原则：

- 优先寻找行业头部品牌、高端品牌或体验成熟的同行网站
- 参考重点是布局、留白、字体、配色、图片气质、节奏、模块组织和 CTA 设计
- 可以借鉴设计语言和页面气质，但不要直接照搬品牌内容或文案
- 原网站只用于提取真实信息和真实链接，不用于决定新版网站的美术方向
- 不要做出明显的 AI 模板感、廉价落地页感或通用 SaaS 官网感
- 设计方向应服务于客户所属行业，而不是套同一套风格到所有项目

默认情况下，AI 应主动拆解参考网站“为什么高级”，再把这些规律迁移到当前客户项目中。

## 真实素材采集原则

对于门店类、本地服务类客户，真实感非常重要。AI 默认应优先使用真实可核实的素材，而不是全部使用虚构或风格照。

默认素材优先级如下：

1. 客户提供的官方素材
2. 客户原网站中的真实素材
3. Google Maps 中与该商家真实对应的照片
4. 客户公开社媒中的真实照片
5. 高质量替代图

执行要求如下：

- AI 可以从客户原网站、Google Maps、Instagram、Facebook 或其他公开渠道中收集真实照片，用于新版网站的筛选、分析和使用
- 使用 Google Maps 或公开社媒照片前，默认先核对是否确实与该商家相关
- 如需核对 Facebook、Instagram 等登录态平台，优先使用用户当前已登录的浏览器会话进行查看，但不要擅自进入账号设置、支付、2FA、安全中心等高风险区域
- 重新核对旧站中的 Facebook、Instagram、Google Maps 入口是否仍然有效，不要盲目沿用失效、跳错或异常跳转的旧链接
- 核实时应尽量结合品牌名、地址、电话、门店环境、品牌陈列、账号命名一致性等信息交叉确认是否属于同一商家
- 如果客户有 Instagram 或 Facebook 链接，默认可以继续进入这些真实链接，挑选其中与门店环境、服务过程、团队状态、顾客体验、空间氛围或真实作品相关的照片
- 如果客户原网站本身就有质量不错、真实感足够强的图片，也可以直接纳入候选素材，不必机械排除原网站图片
- 优先挑选能体现真实环境、团队、服务过程、空间氛围的图片
- 收集到真实图片后，不应全部无差别使用，而应先自行评估清晰度、构图、光线、观感统一性、信息有效性以及是否适合网页展示
- AI 应主动判断每张图更适合放在哪个模块，例如首屏氛围图、品牌展示、门店环境、服务项目、团队介绍、顾客体验、口碑信任区或联系区域
- 如果某张图虽然真实，但画质差、构图乱、信息噪音重、压缩严重、与品牌气质不符，或者放进网站后观感不佳，可以选择不用
- 真实图片优先用于最能建立信任感的区域，例如门店环境、团队介绍、服务现场、品牌展示
- 如果不同来源的真实图片风格差异过大，优先选出一组更统一、更适合形成整体品牌氛围的照片，而不是简单把所有真实图都堆上页面
- 如果真实图质量不足或数量不够，再补高质量替代图，但整体页面仍应尽量保留真实感
- 不要在没有依据的情况下伪造商家环境、团队或服务成果
- 只要客户已有 Google Maps、Instagram、Facebook 或其他关键真实渠道链接，新网站中就应原封不动保留这些链接，不要私自替换、改写或隐藏
- 这些真实渠道入口默认应放在页面最醒目、最容易被用户发现的位置，确保访客能直接进入客户真实账号、地图或原始渠道
- 下载到本地项目中的每张真实素材，默认都应记录最少一份素材说明，至少包含：素材名、来源链接、核实来源、预期使用位置
- 默认不要热链客户的社媒图或地图图作为正式站点素材；如可本地保存并用于展示，应优先下载、归档、命名和记录来源后再接入页面

## 真实渠道与联系方式强化

对于美容院、SPA、本地服务门店、小型工作室等项目，联系方式和真实渠道本身就是信任资产，不应被弱化成页脚补充信息。

默认要求如下：

- WhatsApp、电话、Google Maps、Instagram、Facebook 这类高频入口，应优先放在首屏附近或首页前半段的高可见区域
- 若客户主要靠 WhatsApp 成交或预约，WhatsApp 应保持为第一 CTA，其余电话、地图、社媒作为强辅助入口
- 邮箱通常保留，但视觉层级可略低于 WhatsApp、电话、地图和主社媒入口
- 对于 One-page conversion 类型页面，默认可增加一条醒目的联系方式条、联系卡片或重复 CTA 区块，确保移动端也容易点击
- 如果客户已有多个电话或座机号码，应根据原始公开渠道核实后保留，不要为了简化而擅自删掉真实可用联系方式
- 地址、营业时间、地图链接、电话、邮箱、社媒链接都应确保可点击、可跳转、可在移动端直接使用
- Google Maps、Instagram、Facebook 等真实外链在新站中不应只以图标形式隐蔽出现，必要时可同时提供文字标签提升可见性

## 视觉与前端执行标准

AI 在设计和实现这类网站时，默认必须满足以下标准：

- 以品牌感、专业感、真实感、转化感为优先目标
- 先解决信息层级和页面节奏，再考虑装饰效果
- 不默认做成 SaaS 风、电商风或典型 AI 模板风
- 不使用廉价、堆砌、过度炫技的视觉方式
- 视觉风格应贴合行业和客户定位，可高端、干净、自然、女性向、克制或有质感，但不能空泛
- 页面必须兼顾“好看”和“好转化”，不能只顾设计感而忽视本地商家获客目的
- 如果客户项目属于“单页转化站”，默认应优先保证首屏就能看懂这家店做什么、在哪里、怎么联系、怎么预约
- 真实门店素材一旦足够，Hero、Why Trust Us、Real Experience、Gallery 等核心信任区应优先替换占位图，不应长期保留“只是之后再换”的状态

## 移动端与响应式底线

移动端自适应是硬性要求，不是附加项。

默认要求包括：

- 优先按移动端体验考虑模块组织
- 桌面端和移动端都应完整、可用、美观
- 字号、间距、图片裁切、按钮点击区域都必须适配小屏幕
- 重要 CTA、联系方式、预约入口在移动端不得被弱化
- 页面加载和结构尽量保持轻量，避免展示站变成笨重页面

## 文案与内容重组原则

很多旧网站的问题不只是设计差，也包括内容表达混乱。AI 默认可以对内容进行重组和优化，但应遵守以下原则：

- 可以整理信息结构、重写标题、副标题和卖点表达
- 可以强化服务价值、信任理由和 CTA 的清晰度
- 可以把杂乱信息重组为更适合展示站转化的结构
- 不应凭空编造资质、数据、评价、奖项或商家背景
- 如果存在真实性不确定的内容，应优先向用户确认
- 对于客户已有的 Google Maps、Instagram、Facebook 等真实外链，应保留原始链接地址，不要为了美化页面而省略、改写或替换

## 协作边界与优先级

本手册只定义根目录层面的统一方法，不强制每个客户项目必须采用同一套页面模板或固定文件夹结构。

但在项目组织层面，默认要求保持以下一致性：

- 本地根目录下每个客户一个独立文件夹
- GitHub 仓库 `stoney6379/pages` 中每个客户一个对应文件夹
- 本地项目结构与仓库中的客户文件夹尽量保持一致，方便持续维护和同步更新

默认优先级如下：

1. 当前客户的明确要求
2. 当前客户文件夹内的补充说明
3. 本根目录 `AGENTS.md` 的默认规则

如果客户项目有特殊行业属性、特殊功能需求或特殊审美要求，AI 应优先遵循客户级说明，而不是生硬套用总手册。

## 最终执行原则

AI 处理这类项目时，应始终记住：

- 先问清楚，再开始做
- 先建好客户文件夹，并保持本地与 GitHub 中的结构一致
- 先研究旧站，再决定怎么升级
- 先参考行业顶级设计，再形成新方案
- 先找真实素材，再决定替代图比例
- 原网站、Google Maps、Instagram、Facebook 等真实链接必须原封不动保留，并放在醒目位置
- 先保证移动端和转化逻辑，再追求视觉表现
- 站点代码要及时上传到 `stoney6379/pages` 仓库对应的客户文件夹中

目标不是机械重画一个页面，而是基于客户真实业务，把一个旧的展示站升级成更高级、更可信、更有转化力的新网站。
